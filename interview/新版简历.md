### Super Agent Console｜个人 AI Agent 工程化 Demo｜2026.05

**内容：**

- 技术栈：Nuxt 3、Vue 3、TypeScript、Nitro API、MySQL、Prisma、Docker、K3S、GitHub Actions、GHCR、pino、CLS、火山方舟、豆包 Seed 2.0 Lite。
- 项目目标：用于面试展示和个人能力补齐的轻量 Agent Console，复刻 Agent Runtime、Tool Calling、Trace Console 和可部署运行链路。

**职责 / 实现：**

- 设计双接口 Agent Run 链路：创建消息接口生成 conversation、message、run，再基于 runId 建立 SSE 连接。
- 设计 AgentEvent 协议，统一模型分析、工具调用、Skill 执行、最终回答和异常事件，支持前端 Timeline、数据库落库和 JSON 日志。
- 实现 Tool Schema、Skill Definition、Tool Workflow 和 Model Adapter 配置层，支持 Tool Router、参数校验、workflow input mapping 和 Skill 执行。
- 接入火山方舟豆包 Seed 2.0 Lite，支持流式输出、tool call、Tool Result 回填和最终答案生成。
- 使用 Prisma + MySQL 持久化 Conversation、Message、AgentRun、AgentEvent、ToolCall、SkillRun 和 Feedback。
- 完成 Docker 镜像构建、GHCR 推送、GitHub Actions 自动部署、K3S ConfigMap/Secret 注入、HTTPS 和 CLS 日志采集。
- 设计公网试用保护机制，支持 access code 登录、adminCode 管理入口、Feedback 反馈弹窗、用户维度限频、每日最大 Run 限制、单用户并发 SSE 连接限制及 LLM Kill Switch，避免公网访问导致模型调用成本失控。

**业绩：**

- 打通从用户输入、Agent Run 创建、真实模型 tool call、Tool/Skill 执行、流式输出、数据库落库到线上日志排查的完整闭环。
- 实现轻量版 Agent Trace 能力，使一次 Agent 执行可以按 runId / traceId 复盘模型、工具、Skill 和最终回答过程。
- 补齐个人在 AI Agent 服务端、模型接入、K3S 部署、线上排障和公网 Demo 防滥用方面的工程实践能力。
